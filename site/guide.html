Usage Guide

Using the VortSpec IDE

VortSpec is a desktop cockpit over your own local Claude Code. It configures, launches, observes, and gates Claude Code runs so anyone can run a professional spec-driven design-to-code workflow. It never stores provider keys, proxies no model traffic, and sends no telemetry.

What the IDE is

A VS Code–style workbench: a file Explorer, a code editor, an integrated terminal, a live preview, a visual Run canvas, Source Control, and an assistant panel that is your Claude Code — grounded in the file you have open and always behind spec-first approval gates. The engine is the claude binary already installed on your machine, running under your own login.

Requirements

  • macOS — Apple Silicon or Intel. The IDE ships a separate build for each; pick yours on the download page.
  • Node.js 20 or newer — used to run your project's dev server and installs.
  • Git — for Source Control, cloning, and branch/commit history.
  • Claude Code, installed and logged in — this is the engine. You bring your own Claude; VortSpec adds no account and no keys.
  • Figma — optional, only if you design from Figma sources.

The IDE checks all of these for you on first run (next section) and gives you a one-click fix for anything missing.

Install

1

Download the IDE disk image and drag VortSpec IDE into your Applications folder.

2

First launch. The app isn't notarized yet, so macOS Gatekeeper blocks the first open. Clear it once, either way:

xattr -cr "/Applications/VortSpec IDE.app"

…or right-click the app → Open, then Open again; or System Settings → Privacy & Security → Open Anyway.

3

Open it. You land on the welcome screen where you create, open, or clone a project — or open the bundled walk-through.

NoteSee Troubleshooting → First launch if the app won't open.

First run & what gets validated

VortSpec runs a quick, read-only environment scan. It never changes anything on its own — it only reports status and offers a fix. It checks:

Node.js

Must be version 20 or newer. If missing or too old, VortSpec links you to the download.

Git

Required for Source Control and cloning. Missing → install link.

Claude Code

The engine. VortSpec runs claude --version to confirm it's installed and on your PATH.

Claude Code login

Verified only when you ask (it spends a tiny bit of your Claude usage), via a one-word headless probe — using your own login, never a bare API key.

Figma MCP

Optional. Checked with claude mcp list (no usage). Only needed for Figma design sources — a missing one is informational, not a failure.

Anything that isn't ready shows as a fix-it card with a next step (an install link, an "Open login", or a "Verify" button) — never a raw error. For exactly which of these can stop you and how to clear each, see Troubleshooting → Blockers.

What VortSpec installs for you — with your permission

VortSpec never installs system tools silently. The things it can set up for you, each on an explicit action:

  • Project dependencies. The first time you Run a project with no node_modules, VortSpec runs npm install (or your project's package manager) for you, then starts the dev server. Output streams live so you can see it.
  • A .env file. If your project needs environment variables and ships a .env.example, VortSpec offers a one-click helper to create .env from it — it never invents secret values; you fill those in.
  • The Figma connection (figma-cli). If you use Figma, VortSpec can install and connect figma-cli for you and walk you through enabling fast "yolo" mode. See Connect Figma.
  • The SDD-DE toolkit. Creating a new project scaffolds the Spec-Driven Design Engineering skills (npx @royvillasana/sdd-de) into the project so Claude Code can run the methodology.
Your Claude, your machineEverything happens in the folder you chose, using your own Claude Code login. VortSpec stores no keys and proxies no model traffic.

Open a project

From the welcome screen or the File menu:

  • Create New Project — pick an empty folder and run the guided SDD-DE setup.
  • Open Folder — open an existing project.
  • Clone Repository — paste a Git URL; VortSpec clones it into a folder you choose.
  • Open Walkthrough Project — a complete, correctly-structured SDD-DE reference project to learn from (dependencies install on first Run).
  • Open Recent Projects — jump back into anything you've opened before.

The spec-driven workflow

VortSpec follows the Spec-Driven Design Engineering (SDD-DE) cycle exactly as the CLI does. Generated artifacts — briefs, specs, plans, the design manifest — are gated: nothing advances to implementation, and no downstream files change, until you explicitly approve. You always have a one-click path to the raw form (the file, the terminal) behind every friendly view.

Around the IDE

Editor & Explorer

A lazy file tree with per-type icons and a full code editor. Selections are surfaced to the assistant as grounding context.

Terminal

A real integrated terminal as a tab in the bottom panel — toggle with Ctrl+`.

Live preview & Run canvas

Your running app beside the editor. The Run canvas is a Figma-style layer over the live app — Interact by default; switch to Inspect to edit tokens, spacing, variants, text, and colors visually.

Source Control

Stage, commit, branch, and a GitLens-style commit graph with branch/merge lanes.

The assistant

Your Claude Code with the whole tool surface — it sees your open file and preview, can act on the IDE, always behind the spec-first gates.

Run Doctor

If the dev server won't start, a one-click gated diagnosis uses Claude Code to find and fix the cause. See Troubleshooting.

Connect Figma (optional)

VortSpec's primary Figma link is the local figma-cli, which drives Figma Desktop directly with no token. In the Figma panel you can:

  • Let VortSpec install it if it isn't set up yet.
  • Enable yolo mode (fast, direct Chrome-DevTools patch of Figma Desktop). This needs macOS App Management permission for VortSpec — the panel has a button that opens the exact System Settings pane.
  • Or use safe mode (the MCP bridge / REST token) as a fallback.
OptionalSkip this entirely if you don't design from Figma — it never blocks the IDE.

Where your data lives

Everything lives in your project folder as plain files. Flow state is derivable from those files plus the run log, so any run is cancelable and you can close and reopen mid-flow. Authentication, plan, and usage all belong to your own Claude Code install. VortSpec requires no account, stores no provider keys, and sends no telemetry without opt-in.
